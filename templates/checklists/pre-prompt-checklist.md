# Pre-Prompt Checklist

What to prepare before engaging AI for development tasks.

## Checklist

### Context Preparation
- [ ] Project context is clear and documented
- [ ] Tech stack is specified
- [ ] Architecture patterns are defined
- [ ] Coding conventions are documented
- [ ] Relevant existing code is identified

### Requirements
- [ ] Goal is clearly defined
- [ ] Requirements are specific and testable
- [ ] Constraints are identified
- [ ] Edge cases are considered
- [ ] Success criteria are defined

### Quality Standards
- [ ] Code style requirements specified
- [ ] Testing requirements defined (coverage, types)
- [ ] Security requirements listed
- [ ] Performance expectations set
- [ ] Documentation standards clear

### Resources
- [ ] Similar existing code available for reference
- [ ] Relevant documentation links ready
- [ ] Design decisions documented (ADRs)
- [ ] Test examples available

### Prompt Structure
- [ ] Using structured prompt template
- [ ] Context section complete
- [ ] Requirements section detailed
- [ ] Quality requirements explicit
- [ ] Examples included (if helpful)

## Benefits

✅ Better AI outputs
✅ Fewer iterations needed
✅ Higher quality results
✅ Time savings
✅ Consistent patterns
